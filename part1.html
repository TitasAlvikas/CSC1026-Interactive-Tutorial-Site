HTML versions

The first version of HTML; HTML 1.0 was written by Tim Berners-Lee in 1993 and was a very simple langauge used to handle text-based documents. The next versions of HTML; HTML 2.0 up to HTML 3.2 standardised the features of the language by adding in forms, tables, applets (java programs used in a webpage) and text flow around images. In 1999 HTML 4.01 was released giving support for CSS (Cascading Style Sheets) styling, Javascript, imporved document rendering and standardised support for multimedia.

XHTML

XHTML 1.0 was created to tighten up on the HTML syntax using XML (Extensible Markup Language) markup rules. The tags and attributes used in XHTML are the same as in HTML 4.01 with the only differences being in the structure and syntax of the markup language. These differences are:

  • XHTML elements must be properly nested.
  • XHTML elements must always be closed.
  • XHTML tags and attributes must be in lowercase.
  • Attribute values must be quoted.
  • XHTML DOCTYPE declarations are mandatory.

XML

XML is a standardised specification of creating new markup languages. XML and HTML were derived from SGML (Standard Generalized Markup Language). Many document formats have been developed using the XML syntax including RSS, SVG and XHTML. The design goals of XML focus on simplicity, generality and usability across the web.

Advantages and Disadvantages of XHTML

Advantages:

  • XHTML allows for increased portability of webpages across wireless devices and other specific web environments.
  • Pages written in XHTML display more predictably in browsers.
  • XHTML webpages can be easily converted to other file formats such as PDF and RSS.

Disadvantages:

  • Developers writing XHTML are prone to making easy mistakes with the strict markup.
  • Currently there are not enough browsers that are compatible with XHTML (Microsoft Internet Explorer does not properly support XHTML).

HTML5

After developing HTML 4.01 and XHTML 1.0, the people in charge of creating new HTML versions went on to create the new version of HTML5 which introduces some new tags and attributes (mainly <video>, <audio> and <canvas> tags) and allows you to describe the semantics of a website's content. HTMl5 reduces the need for browser plugins to use interactive content and provides compatibility to use the new CSS3 version of CSS. This latest version of HTML also provides extentsion to the Document Object Model (DOM).

Which version of HTML should you choose?

Which ever language you choose to use to develop a website with you must make sure that the markup validates with the current standards for that language on one of these official validator websites:

http://validator.w3.org/

http://validator.nu/

HTML: Basic document structure

The W3C website outlines the basic structure of a HTML document:

  1. A line containing HTML version information
  2. A declarative header section (delimited by the HEAD element)
  3. A body, which contains the document's actual content. The body may be implemented by the BODY element or the FRAMESET element.

Sections 2 and 3 should be delimited by the <html> element.

DTD declaration

The DTD (Document Type Declaration) defines which version of HTML should be used and how the web browser should render the markup.
An example of a DTD declaration for a HTML 4.01 web document:

<!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.01//EN"
"http://www.w3.org/TR/html4/strict.dtd">

The PUBLIC part identifies which version of HTML should be used. The URL defines the location of the markup specification.

Header section

The declarative header section includes a <title> tag and which character set to use. Some optional declaratins include a <meta> tag for metadata, <script> tags for javascript files, <style> and/or <link> tags for cascading style sheets.
Note: A header section NEVER contains any content.

Body section

<body>
<div id="title">
<h1>Sign Language</h1>
<ul id="navbar">
<li><a href="#Introduction">Introduction</a></li>
</ul>
</div>
<div class="normalbox">
<h2><a id="Introduction">Introduction</a></h2>
<p>Sign language is a language that uses manual communication to convey meaning.</p>
<hr />
</div>
</body>

The body section contains the content that the user sees when loading up a webpage. The appearance of it may change from the information declared in the header such as scripts and style sheets.

Tags

An HTML tag consists of an opening tag and an ending tag.

<h1>A heading</h1>

Some tags dont display any content.

<br /> <hr />

Block level tags can contain nested inline tags.

<p>The <strong>bold</strong> part of a paragraph</p>

Attributes

All HTML tags can have attributes. They are always declared in the opening tag. They usually come in name/value pairs (name="value"). Every attribute in XHTML must have a value. The most common attributes which can be used on any body tag are class, id, style and title.

<a href="https://www.w3schools.com">A link</a>

checked -This is incorrect

checked="checked" -This is correct

Block-level tags

Heading tags can have 6 different levels (only one h1 tag per page) and levels cannot be skipped.

<h1>A heading</h1>

<h1> <h2> <h3> -This is correct

<h1> <h2> <h3> <h2> -This is correct

<h1> <h3> -This is incorrect

<h2> <h3> -This is incorrect

Paragraph tags are used as containers for body text.

<p>A block level paragraph</p>

Line breaks can be used but should be avoided.

<p>A line<br />Break</p>

A horizontal line can be created using the <hr> tag.

<p>Section 1</p> <hr /> <p>Section 2</p>

There are 3 different types of lists; Unordered, Ordered and Definition lists.

Unordered list (bullet points)

<ul>
<li>Bullet Item</li>
<li>Bullet Item</li>
<li>Bullet Item</li>
</ul>

Ordered list (numbers)

<ol>
<li>Item 1</li>
<li>Item 2</li>
<li>Item 3</li>
</ol>

Definition list (term and definition)

<dl>
<dt>List</dt>
<dd>A collection of items</dd>
<dt>Tag</dt>
<dd>A type of HTML element</dd>
</dl>

Inline tags

The most common inline tags include <em> (for emphasis), <strong> (gives text in bold), <cite> (for citations or references), <code> (for program code in a monospace font).

Links can be added using the <a> tag. The location of the link is placed in the href attribute.

To link to other websites:

<a href="www.wikipedia.com">Wikipedia</a>

To link to other pages on a local site:

<a href="mysite.html">My website</a>

To link to other parts of a webpage:

<a href="mysite.html#section5">Section 5 of my website</a>

Tables

Tables are only used to display tabular data in webpages (can be enhanced using CSS).

<table>
<tr>
<th>Country</th><th>Population</th>
</tr>
<tr>
<td>USA</td><td>200m</td>
</tr>
<tr>
<td>Japan</td><td>10m</td>
</tr>
</table>

The most common table attributes are border (controls outside border properties), width (width of table), cellspacing (controls distance between two cells), cellpadding (controls distance between cell content and cell egdes).

For more detailed information on tables visit this link at W3Schools.

Structural grouping

To group some block-level elements together the <div> tag can be used with class and id attributes to add styles and scripts to single blocks.

To style inline fragments of content the <span> is used to style lines of text or single characters.

<div id="section1" class="title">
<h1>Heading</h1>
<p>Paragraph</p>
</div>

<p>The <span>bold</span> part of paragraph</p>

Images and Multimedia

An image can be loaded on a webpage by using the <img> tag with the src attribute acting as the URL location of the image and a required alt attribute that provides a description of the image if it fails to load. The width and height attributes are used to provide the dimanesions of images.

<img src="LocationOfImage.png" alt="wikipedia logo" />

For other multimedia objects, the <object> and <param> tags are the most universal way of displaying media objects. Previously, the <embed> tag had been widely used but is now starting to phase out.

<object width="300" height="240"
type="application/x-shockwave-flash"
data="http://videoURL.tv">
<param name="movie" value="http://videoURL.tv" />
<param name="allowFullScreen" value="true" />
<param name="allowScriptAccess" value="always" />
</object>

The HTML 5 specification has the updated and improved video and audio tags to play multimedia.

<video width="300" height="240"
src="videoURL.ogg"
type="video/ogg"
controls="controls">
</video>
