HTTP Requests

HTTP request-response protocol

The HyperText Transfer Protocol is the foundation for data communication on the World Wide Web. A client sends a HTTP request message to ask for resources from a server. Then the server answers with a response message and any content to be dispalayed. HTTP messages are composed of textual information encoded in ASCII. HTTP requests and responses are transferred by TCP/IP packets. A request message contains metadata in it's header and data in an optional body. Each HTTP request is exchanged independently.

HTTP methods

The HEAD and GET methods are both mandatory methods, GET requests a specified resource and HEAD asks for a response identical to that of GET but without the response body. The HEAD, PUT, GET and DELETE methods are all idempotent meaning that the effect of multiple requests is the same as for a single request.

HTTP caching

HTTP caching occurs when a browser stores local copies of external resources for quick retrieval of these resources. This process eases the load on the server as it is not required to transmit the resources again and the browser is able to load webpages quickly the next time they are reloaded.

HTTP request structure

An example of a basic HTTP request is shown here:

Request line: GET /www.wikipedia.com/index.html HTTP/1.1
General headers: Date: Tue, 23 Nov 2017 12:53:00
Request headers: Host: b234359.ncl.ac.uk..., User-Agent: Google Chrome 68.0
Entity headers: empty except for POST and PUT methods
Message body: empty except for POST and PUT methods

HTTP response structure

An example of a basic HTTP response is shown here:

Status line: HTTP/1.1 200 OK
General headers: Date: Tue, 23 Nov 2017 12:57:00
Response headers: Server: Apache/2.0
Entity headers: Content-Length: 7398, Content-Type: text/html
Message body:
<!DOCTYPE html>
<html>
HTML webpage content here
</html>

Web Clients

A web client or browser is an application that provides communication to the web by using the HTTP protocol to load, display and send data to other web servers connected to the internet. Web browsers contain a user interface that allows users to navigate the web with back, forward and reload buttons, they allow users to view the websites that a user has visited and they allow a user to manage their downloads from the web. The most common web clients and widely known are:

  • Google Chrome
  • Mozilla Firefox
  • Internet Explorer
  • Safari
  • Microsoft Edge
  • Opera
